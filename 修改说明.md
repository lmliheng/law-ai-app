# 修改说明（本地部署与“得理/腾讯元器”接入）

本文档记录：从最开始的本地部署开始，为了跑通项目并接入“得理开放平台 / 腾讯元器”，本仓库内**新增/修改**了哪些文件，以及每个文件的作用。

## 1. 后端（`Back/node`）

### 1.1 修改：`Back/node/.env`
- **改动内容**
  - 修正/补充 MySQL 连接配置（`DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`）
  - 新增得理开放平台密钥配置：
    - `DELI_APPID`
    - `DELI_SECRET`
  - 新增腾讯元器智能体 API 配置：
    - `YUANQI_ASSISTANT_ID`（智能体 appid，**不是体验链接**）
    - `YUANQI_APPKEY`（API 管理里的 appkey）
    - `YUANQI_API_BASE`（默认 `https://yuanqi.tencent.com`）
- **目的**
  - 让后端可连接本地 MySQL，并支持后端代理调用得理/元器接口（避免前端泄露密钥）。

### 1.2 修改：`Back/node/server.js`
- **改动内容**
  - 挂载新增路由：
    - `app.use('/api/deli', require('./routes/deli'))`
    - `app.use('/api/yuanqi', require('./routes/yuanqi'))`
  - 调整 CORS：允许前端通过本地静态服务器（`http://127.0.0.1:8000` / `http://localhost:8000`）访问后端；兼容 `file://` 打开时 `Origin` 为空的情况。
- **目的**
  - 把“得理/元器”能力接入到现有 Express 服务中，并让前端能在本机正常调用后端接口。

### 1.3 新增：`Back/node/routes/deli.js`
- **提供接口**
  - `POST /api/deli/cases`：类案检索（对应手册 `queryListCase`）
  - `POST /api/deli/laws`：法规检索（对应手册 `queryListLaw`）
- **鉴权方式**
  - 通过请求头携带 `appid`、`secret`（从 `.env` 读取）
- **目的**
  - 后端直连得理开放平台，统一对前端提供本地接口，避免在浏览器端暴露得理密钥。

### 1.4 新增：`Back/node/routes/yuanqi.js`
- **提供接口**
  - `POST /api/yuanqi/chat`：将用户问题转发给腾讯元器智能体 API（`/openapi/v1/agent/chat/completions`）
- **鉴权方式**
  - `Authorization: Bearer <YUANQI_APPKEY>`（从 `.env` 读取）
- **目的**
  - 推荐方案：把“得理检索 + 总结输出”的工作流配置在元器智能体中，本项目只需要调用元器 API 即可获得最终回答。

## 2. 前端（`Font/JS`）

### 2.1 修改：`Font/JS/index.html`
- **改动内容**
  - 增加/修正导航指向 `./model/chat/chat.html`（问答页面入口）
  - 移除原 Coze Web SDK 的直接加载引用，避免与“腾讯元器方案”混用
- **目的**
  - 让首页能进入新的“腾讯元器问答页”，并避免同时加载无关第三方聊天 SDK。

### 2.2 新增：`Font/JS/model/chat/chat.html`
- **用途**
  - 提供一个简洁的问答页面 UI（输入框、消息列表、发送按钮）。

### 2.3 新增：`Font/JS/model/chat/chat.js`
- **用途**
  - 调用本机后端接口 `POST http://127.0.0.1:3000/api/yuanqi/chat`
  - 解析元器返回的 `choices[0].message.content` 并渲染到页面

### 2.4 新增：`Font/JS/model/chat/chat.css`
- **用途**
  - 为问答页面提供基础样式（深色背景、消息气泡、输入区等）。

## 4. 运行方式（快速）

### 4.1 后端
在 `Back/node/.env` 配好数据库与元器参数后：

```bash
cd Back/node
npm install
npm start
```

### 4.2 前端
建议用静态服务器启动：

```bash
cd Font/JS
npx http-server -p 8000
```

浏览器打开：`http://127.0.0.1:8000/index.html`  
问答入口：`/model/chat/chat.html`
