# 法律AI智能应用系统

## 项目简介

本项目是一个面向法律领域的智能化应用系统，旨在通过人工智能技术提升法律工作的效率和质量。系统集成了智能体开发、工具应用和系统解决方案三大核心能力，为法律从业者、企业和个人用户提供全方位的智能化法律服务。

### 1. 智能体开发
- **法律咨询智能体**：基于法律专业知识库，提供7×24小时智能法律咨询服务
- **案件分析智能体**：自动分析案件材料，识别关键法律问题，提供案件预判建议
- **合规审查智能体**：针对企业业务流程进行合规性检查，识别潜在法律风险

### 2. 工具应用开发
- **法律文书自动生成**：根据用户需求自动生成各类法律文书，包括合同、起诉状、答辩状等
- **案例智能检索**：基于语义理解的智能案例检索系统，快速找到相关判例和法条
- **合同条款审查**：自动审查合同条款，识别风险点，提供修改建议
- **证据材料分析**：对证据材料进行智能化整理、分类和分析，提取关键信息

### 3. 系统解决方案
- **智能法律援助平台**：为公众提供便捷的法律援助服务，包括在线咨询、案件受理、进度跟踪等
- **企业合规管理系统**：帮助企业建立完整的合规管理体系，实现合规风险的识别、评估和监控
- **司法辅助决策系统**：为司法工作者提供案件分析、法律条文检索、类案推荐等辅助决策功能

## API调用格式与注意点

### 腾讯元器API调用格式

#### 1. API配置信息

```javascript
const API_CONFIG = {
    appkey: 'YRsdNKlaJJU5YibO0OGbYzKWwaklOKSn',
    appid: '2035707181828373248',
    apiUrl: 'https://yuanqi.tencent.com/openapi/v1/agent/chat/completions',
    stream: true
};
```

#### 2. 请求格式

```javascript
const requestBody = {
    assistant_id: API_CONFIG.appid,
    user_id: 'user_' + Date.now(), // 使用唯一用户ID
    stream: API_CONFIG.stream,
    messages: messagesForAPI
};
```

#### 3. 消息格式

```javascript
// 用户消息格式
const currentUserMessage = {
    role: 'user',
    content: [
        {
            type: 'text',
            text: message
        }
    ]
};

// 助手消息格式
const assistantMessageObj = {
    role: 'assistant',
    content: [
        {
            type: 'text',
            text: assistantMessage
        }
    ]
};
```

#### 4. 发送请求

```javascript
const response = await fetch(API_CONFIG.apiUrl, {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${API_CONFIG.appkey}`
    },
    body: JSON.stringify(requestBody)
});
```

### 注意事项

#### 1. 消息格式验证
- **第一条消息必须是user**：API要求对话的第一条消息必须是用户消息
- **角色必须交替**：消息角色必须严格按照user→assistant→user→assistant的顺序交替
- **内容格式**：消息内容必须是包含type和text字段的数组格式

#### 2. 流式响应处理
- **流式响应**：设置`stream: true`时，需要处理流式数据
- **数据解析**：流式响应需要逐行解析SSE格式数据
- **错误处理**：需要捕获并处理流式解析过程中的错误

#### 3. 对话历史管理
- **初始化**：对话历史初始化为空数组，不要包含任何初始消息
- **更新时机**：只有在收到助手实际回复后才更新对话历史
- **消息顺序**：确保对话历史中的消息角色严格交替

#### 4. 错误处理
- **网络错误**：捕获并处理网络请求失败的情况
- **API错误**：处理HTTP状态码非200的情况
- **格式错误**：验证消息格式，避免因格式问题导致API调用失败

#### 5. 用户体验
- **加载状态**：在API请求过程中显示加载状态
- **输入禁用**：发送消息后禁用输入直到收到响应
- **错误提示**：显示友好的错误提示信息
